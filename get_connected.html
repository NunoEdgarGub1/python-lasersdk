Get Connected with Your LaserÂ¶

To connect to a laser device via Ethernet or USB/RS232 an instance of the toptica.lasersdk.client.Client class is required. The instantiation operation of the toptica.lasersdk.client.Client class takes a toptica.lasersdk.asyncio.connection.Connection object as its argument.

The class toptica.lasersdk.asyncio.connection.Connection is an abstract class from which the two classes toptica.lasersdk.asyncio.connection.NetworkConnection and toptica.lasersdk.asyncio.connection.SerialConnection are derived.

Note

Python 3.7 changed async and await to reserved keywords. Since version 2.0 of the SDK every module starting with toptica.lasersdk.async.* has therefore been renamed to toptica.lasersdk.asyncio.*.

The instantiation operation of toptica.lasersdk.asyncio.connection.NetworkConnection requires a host identifier as first argument. This can be an IP address, a symbolic hostname that can be resolved using the domain name system (DNS), or a system label that can be resolved by the DeCoP discovery service. The latter is a custom device label that can typically be set by the end user using the system-label DeCoP parameter.

Network example:

from toptica.lasersdk.client import NetworkConnection

conn1 = NetworkConnection('10.0.0.2')
conn2 = NetworkConnection('laser.mycompany.com')
conn3 = NetworkConnection('My Laser')

The instantiation operation of toptica.lasersdk.asyncio.connection.SerialConnection requires a serial port identifier as first argument.

Serial port example:

from toptica.lasersdk.client import SerialConnection

conn1 = SerialConnection('COM1')         # On Windows
conn2 = SerialConnection('/dev/ttyUSB0') # On Linux
